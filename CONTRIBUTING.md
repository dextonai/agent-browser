# Contributing to agent-browser

Thank you for your interest in contributing to `agent-browser`! This guide will help you get started with our development environment and submission process.

## Development Environment Setup

1. **Clone the repository:**
   ```bash
   git clone https://github.com/dextonai/agent-browser
   cd agent-browser
   ```

2. **Install dependencies:**
   This project uses `pnpm`.
   ```bash
   pnpm install
   ```

3. **Install Playwright browsers:**
   ```bash
   pnpm exec playwright install --with-deps chromium
   ```

4. **Build the project:**
   ```bash
   pnpm build
   ```

5. **(Optional) Build the native Rust CLI:**
   If you want to contribute to the Rust CLI, you'll need the [Rust toolchain](https://rustup.rs) installed.
   ```bash
   pnpm build:native
   ```

## Coding Style

- **TypeScript:** We use Prettier for code formatting. You can run `pnpm format` to format your code.
- **Linting:** We use ESLint and Oxlint. Run `pnpm typecheck` to verify types.
- **Commits:** We follow conventional commit messages (e.g., `feat:`, `fix:`, `docs:`).

## Testing

Before submitting a PR, ensure all tests pass:
```bash
pnpm test
```

For the Rust CLI:
```bash
cargo test --manifest-path cli/Cargo.toml
```

## Submitting a Pull Request

1. Create a new branch for your feature or fix.
2. Ensure your code follows the style guidelines and tests pass.
3. Submit a PR against the `main` branch.
4. Provide a clear description of the changes in your PR body.
